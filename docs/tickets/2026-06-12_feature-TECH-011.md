### [Feature] Add SCS service modules (user, layout, campaign, template)

**Story:** As a developer, I want four self-contained Spring Boot microservice modules each with a HAL root controller, layered YAML configuration, and Dockerfile, so that each domain service can be built, tested, and deployed independently.

**Acceptance Criteria:**
- [ ] Each module has a `pom.xml` with parent pointing to root aggregator and `spring-boot-maven-plugin` declared
- [ ] Package convention follows `com.voidfuldawn.encountersofthevoid.{user|layout|campaign|template}`
- [ ] Each module exposes `GET /api/{resource}/` returning HAL `CollectionModel` with HTTP 200
- [ ] Services run on ports 8081 (user), 8082 (layout), 8083 (campaign), 8084 (template)
- [ ] `application.yaml` includes `spring.application.name`, H2 in-memory datasource, and correct port
- [ ] `application-prod.yaml` uses `${DB_URL}`, `${DB_USERNAME}`, `${DB_PASSWORD}` and disables H2 console
- [ ] `application-test.yaml` is located under `src/test/resources/`
- [ ] H2 dependency scope is `test` in each module POM
- [ ] Multi-stage Dockerfile uses `eclipse-temurin:21-jdk-alpine` builder with `./mvnw` and `eclipse-temurin:21-jre-alpine` runtime

**Resolution:** Four Spring Boot modules were created under the repository root with `Application.java` entry points, HAL controllers using `WebMvcLinkBuilder`, and a three-profile YAML configuration strategy. Each module's Dockerfile copies the Maven wrapper and parent POM into the build context to avoid requiring a system Maven installation.

### [Feature] Add Gateway module

**Story:** As a developer, I want a Spring Cloud Gateway module that routes all four `/api/**` path prefixes to the correct backend services, so that clients have a single ingress point.

**Acceptance Criteria:**
- [ ] `gateway/pom.xml` depends on `spring-cloud-starter-gateway`, `spring-boot-starter-actuator`, and `spring-boot-starter-security` (no `spring-boot-starter-web`)
- [ ] `application.yaml` routes `/api/users/**` → `http://user-service:8081`, `/api/layouts/**` → `http://layout-service:8082`, `/api/campaigns/**` → `http://campaign-service:8083`, `/api/templates/**` → `http://template-service:8084`
- [ ] `management.endpoints.web.exposure.include: health` is set
- [ ] A permissive `SecurityWebFilterChain` bean permits all proxied requests without requiring authentication

**Resolution:** The gateway module was created with reactive Spring Cloud Gateway configuration. Service URIs were set to Docker Compose service-name hostnames (not `localhost`). A `SecurityConfig` class declaring a permit-all `SecurityWebFilterChain` was added to prevent Spring Security's default 401 behaviour from blocking proxied requests.
